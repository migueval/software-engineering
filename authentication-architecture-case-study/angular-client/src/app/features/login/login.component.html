Caso de Estudio Técnico

Authentication Architecture Case Study

Una comparación interactiva de arquitecturas de autenticación modernas. Este proyecto demuestra cómo una misma lógica de negocio se comporta operativa y estructuralmente al implementarse como un monolito tradicional o como un sistema distribuido de microservicios.

📦

Arquitectura Monolítica

Implementación centralizada en una única base de código NestJS. Conexión directa a PostgreSQL mediante TypeORM. Ideal para simplicidad, transacciones atómicas rápidas y mínimo retardo de red.

  • Base de datos relacional unificada
  • Llamadas síncronas en memoria
  • Despliegue y mantenimiento simplificado
⚙️

Arquitectura de Microservicios

Estructura modular desacoplada mediante un API Gateway. Utiliza RabbitMQ como broker de mensajería asíncrona para notificaciones y Redis para almacenamiento rápido de sesiones y OTP.

  • Aislamiento estricto de fallos
  • Escalabilidad horizontal independiente
  • Mensajería por colas (RabbitMQ) y caché (Redis)

💡 Usa el selector de la izquierda para alternar la infraestructura de destino y comparar los flujos en tiempo real.

Technical Case Study

Authentication Architecture Case Study

An interactive comparison of modern authentication architectures. This project demonstrates how the exact same business logic behaves operationally and structurally when implemented as a traditional monolithic system or as a decoupled, distributed microservices network.

📦

Monolithic Architecture

Centralized implementation in a single NestJS codebase. Direct connection to PostgreSQL database using TypeORM. Best suited for simplicity, fast atomic transactions, and minimal network latency.

  • Unified relational database storage
  • In-memory synchronous calls
  • Simplified deployment and operations
⚙️

Microservices Architecture

Decoupled modular structure utilizing an API Gateway. Employs RabbitMQ as an asynchronous message broker for notifications and Redis for high-speed session and OTP caching.

  • Strict fault isolation
  • Independent horizontal scalability
  • Queue-based messaging (RabbitMQ) & cache (Redis)
